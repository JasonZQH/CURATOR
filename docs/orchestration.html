The control loop, and its boundary

Curator's Phase 0 turns an accepted goal into a durable, single-writer loop: a deterministic scheduler — not the model — owns every transition, and each step lands on a replayable SQLite ledger. This is where that loop is drawn, and where the current implementation ends and the V1 blueprint begins.

W What

A local control plane, not a chat wrapper

The repository implements a local Phase 0 control plane: an accepted goal becomes a durable SQLite workflow session containing a writer run, deterministic verification, a fresh-context review, and a human confirmation gate. The Excalidraw blueprint describes the next runtime model — it is not a claim that the current branch already has a task DAG or concurrent workers.

H How

The current Phase 0 path

Each stage in Figure 1 is owned by a specific module. The scheduler resolves a provider driver per slot binding, packages context, and pauses rather than guessing whenever a precondition is missing.

shell/repl.py
  · records discussion, approval, provider setup, and resume input
app.py
  · creates accepted-goal workflow sessions
loops/compiler.py
  · compiles the fixed single-writer delivery template
scheduler/engine.py
  · builds context packages and runs the sequential plan
  · resolves provider drivers from slot bindings
  · pauses on missing provider, dirty workspace, handoff, or missing verification
providers/claude_code.py · providers/codex_cli.py
  · render context packages into real CLI prompts
  · stream provider events and capture workspace evidence
harness/verifier.py
  · discovers project verification commands and records deterministic evidence
harness/workspace.py
  · requires a clean git baseline for writer runs
  · records tracked diff and untracked files after provider execution

The blueprint target boundary

The target separates business execution from a provider session. A durable Execution owns one task attempt; every provider process or native TUI session is an AgentRuntime, recorded by an ExecutionRuntime. A runtime handoff keeps the same execution and attempt number while appending a new runtime sequence. A user retry creates a new execution linked by retry_of_execution_id and preserves the interrupted record.

The target scheduler owns Project, Session, Goal, Task, Dependency, Agent, Provider, Capability, Artifact, Memory, Approval, lease, heartbeat, and event state. It may run independent ready tasks in parallel only after workspace ownership, permission scope, concurrency limits, and recovery semantics exist. Control Desk is the inspection/control surface; the concrete Claude Code or Codex TUI remains the native execution surface.

Provider setup accepts only claude-code and codex in Phase 0. Detection requires the CLI binary to exist and --version to exit successfully; broken CLIs are never persisted as active profiles. The blueprint's capability detection, authentication health, version compatibility, installation registry, and native TUI lifecycle remain V1 work.

Y Why

Auditable without pretending chat is the record

The boundary makes real provider behavior auditable without treating a provider conversation as the system of record. Phase 0 already makes setup requirements, typed failures, verification evidence, pauses, and resume choices visible. The blueprint extends the same principle to task state transitions, capability invocations, approvals, artifacts, runtime leases, heartbeats, and handoff snapshots.

F Future · considerations · trade-offs

Migration order

  • Introduce the canonical Project/Goal/Task/Execution schema beside the Phase 0 ledger.
  • Add append-only events, worker leases, heartbeats, restart recovery, and a bounded retry policy.
  • Add filesystem-backed artifact and memory manifests; keep large logs and patches out of SQLite.
  • Define Provider Adapter and shell adapter contracts before adding tmux, Ghostty, or Warp integration.
  • Add capability install/status records and STRICT, BALANCED, AUTONOMOUS, and FULL_ACCESS policy evaluation.
  • Only then enable DAG scheduling and up to four concurrent local workers.
